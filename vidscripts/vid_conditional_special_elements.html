1. Introduction

Top

  • Hi, I'm Ali Hussein, creator of Entmorph,
  • and in this video we're going to introduce Special Elements,
    • which are an Entmorph extension to HTML
    • which allows us to make decisions about what to display in the User Interface based on changes in reactive variables;
  • As usal the best way to understand this is to see it demonstrated;
  • There are several different Special Elements,
    • so in this video we'll explore Conditional Special Elements,
    • and in the next video we'll look at Sequence Special Elements;
  • We're going to create this component, which has two sections;
    • at the top, there's a form requesting you to login using your username;
    • if we click on the checkbox, the form switches so that you can login using your email address instead;
  • Then underneath that is a checkbox which if we check it will show a set of advanced controls for a widget;
  • These are both just examples which demonstrate how Special Elements work, and as such they don't do anything at all;
  • So let's get started;

4.3. Add :#id email-checkbox

Top

  • The :label element :for attribute needs an id for the checkbox;
  • In HTML, we would set the :id attribute of the checkbox,
    • :id "email-checkbox"
  • But if we did this here, if we want to create more than one instance of this component on the same page,
    • each instance would have a checkbox with an id of "email-checkbox",
    • which isn't allowed;
  • To solve this, we'll use a Special Attribute;
    • click on Add Special > :#id
  • This is an Entmorph extension to HTML,
    • and what this does is each time a new instance of this component is created,
    • it generates a new id for the checkbox,
    • and that id is guaranteed to be unique;
  • :#id email-checkbox
  • Even though we've called in "email-checkbox" here, this is just so we can refer to it, for example from another attribute;

6. Add :div.section

Top

  • Now let's add a section which will respond to changes in the email-login? state variable, which the checkbox is updating;
  • scroll-to div.checkbox-container
  • [close (div.checkbox-container) ]
  • Underneath it we'll add:
  • :div.section
  • click on Special
  • which opens a sub-menu with the special elements which are currently available;
  • click on :#if
    • which creates an :#if special element,
    • which has 3 sections:
      • Condition
      • True Element, which is rendered when the Condition is true,
      • False Element, which is rendered when the Condition is false
  • We'll start with the Condition
  • This can be an expression, including an @Reactive;
  • click on nil > @Reactive > email-login?
  • See following diagram:
[:div.section
 [:#if @email-login?
  ;; Open True Element
  [:div
   "Login using email address"
   ;; Leave :value empty
   [:input/text
    {:placeholder "E-mail"}]
   ;; Leave :on-click empty
   [:input/button
    {:value "Login"}]]
  ;; Copy and paste from True Element
  [:div
   "Login using username"
   [:input/text
    {:placeholder "Username"}]
   [:input/button
    {:value "Login"}]]]]

8. Review of :#if

Top

  • scroll-to :#if
  • So the :#if Special Element,
    • through its Condition expression,
    • reacts to changes in the @email-login? state variable,
    • and renders the True or False Element branch appropriately;
  • The :#if Special Element is very useful if you have two alternatives and you want to switch between them;
  • However there are also times when you want to be able to show or hide something;
  • For that, we need to use the :#when Special Element

11. Summary

Top

  • In this video we looked at two conditional special elements,
    • :#if, which lets us choose what to display between two alternatives;
    • :#when, which lets us show or hide a part of our User Interface
  • Each of these Special Elements lets us set a Condition expression,
    • which controls what is rendered in the User Interface
  • We can set anything here, but if we use an @Reactive, we make the whole Special Element react to changes to reactive variables;
  • There's also a :#cond Special Element which will be released soon, which allows us to choose from multiple conditions and alternatives;
