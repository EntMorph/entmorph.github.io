4. Brief Overview

  • In the Code Editor, we create what are called cells;
  • Each cell contains code, which we can then evaluate to produce an output;

5. Namespace Commands

  • scroll-to Namespace Commands
  • which gives us options to create 3 types of cell:
    • Data and Actions, which are covered in this and the next video,
    • Components, which are covered in a separate series of videos
  • click on Add Data
    • which opens a sub-panel with a set of data types we can add;

6. Add Number

  • click on Number
  • This creates a Number cell,
    • which has a number of sections;
  • At the very top, there's an autogenerated Id,
    • which we can use to refer to the cell from within other cells;
  • It's a good idea to rename a cell's Id as soon as you create it;
    • click on Id
    • Rename: my-number
  • Underneath that, there are Undo, Redo and Delete buttons,
  • followed by a Cell Commands panel,
    • which we'll look at later;
  • Then, there's a Code Panel, which is the most important part of the cell;
    • Since we selected Number, there's a Number editor in the Code Panel
    • I can set its value in the textbox: 12345
    • and update it by selecting any of the commands in the Number Commands panel above it;
  • Scroll so Output Panel is in view
  • Below the Code Panel, there's a Run Panel,
    • which contains a Run Button,
    • and a Live Edit checkbox;
  • click on Run
    • the Output Panel becomes populated with the Number

7. Close Cell

  • Once we're done editing, it's a good idea to close an editor to keep the User Interface tidy;
  • If I click on Close next to the Number editor heading,
    • both the Number commands and textbox disappear,
    • and we just see the number itself;
  • We can open the Number editor again by clicking on it;
  • [open Number for editing]
  • If we scroll-to Cell Id
    • and click on Cell Id : Close
  • This closes the whole cell for editing,
    • including the Number editor,
    • and it hides the Cell Commands;

14. Add Map

  • Now let's add a Map
  • Rather than go back to the Namespace Commands to add another cell,
    • we can use the Cell Commands at the top of each cell
    • to insert a new cell before or after the current cell;
  • click on Insert After > Map
  • Which creates a Map cell after the current cell;
  • scroll-to {{{Cell Id}}}
  • Rename Id: my-map
  • Map is short for Hash-map, also known as a dictionary or an associative array in some languages;
  • It's a data structure which lets you store a unique key with an associated value;
  • We can then use that key to look up the value in the map;
  • You can use any type of value as the key for a map,
    • and its very common to use strings and numbers,
  • But even more common is to use keywords as keys, which is why there's a default empty keyword which is created when you first create a new Map;
  • Let's add some keys to it;
{:number my-number
 :string my-string
 :vector my-vector}
  • click on Run

  • There are a lot of things we can do with them, and here we'll look at a small handful of operations.

  • Review

  • click on Live Edit
  • Change :number -> :string -> :vector
  • click on Live Edit

15.4. assoc

  • Next we'll look at how to add a value to a map;
  • click on Cell Commands > Insert Action After > Call
  • Rename Id: assoc-map-action
  • Select: assoc
    • which creates a call to the assoc function;
  • This function takes a minimum of 3 arguments,
    • which we'll add now;
  • The first is the collection we want to update;
  • The second is the key within that collection whose value we want to update;
    • click on Keyword > :number
  • And the final argument is the value that we want to give the key within the map;
    • click on Number > 54321
  • click on Run
  • Note change in just :number key
  • This is a good time to mention that Entmorph uses immutable data structures,
    • and this applies to maps, vectors and as we'll soon see, sets;
  • What this means is you can't update any of these structures,
    • at least not in the traditional sense;
  • Instead, when we call a function like assoc,
    • what it actually does is it creates a new map which contains the change you want;
  • But the original map doesn't change, and what this means is that both the original and the new map exist at the same time;
  • This has many implications, the overwhelming majority of them good,
    • and we'll go over these in future videos as well as in the documentation;

17. Automatic Reference Updating

  • Goto > Top > ~my-number~
  • One very useful feature of the Code Editor which I want to show you is this;
  • Suppose we want to rename my-number to another-number;
  • In a traditional text-based code editor, we would have use the text editor's search and replace operation, and manually go through all occurrences of the search string to confirm the replacement;
    • even in a single file it can be quite tedious,
    • but if the item we're renaming exists in multiple files,
    • then the process can be very time-consuming and prone to errors;
  • In Entmorph, all we have to do, is to rename it;
  • Rename my-number -> another-number
  • If we scroll-to Goto Cell
    • it's reference has automatically been updated;
  • click on Goto Cell > my-vector
    • where it's also been updated
  • click on Browser Back > Goto Cell > my-map
    • where again, it's also been updated to another-number
  • This is a general feature of Entmorph editors;
    • and doesn't just apply to cell ids;
  • Anywhere you create an id, if you ever rename it,
    • all references to that id will automatically be updated;

  • If I click on Undo, that should reverse the renaming of my-number,

  • First, the Redo button has now been enabled;
  • If we look at the Undo History,
    • the highlighted row has moved up to the previous row;
  • Presing Undo again will reverse the that change,
  • But pressing Redo instead will cause the Rename change to be redone;
  • click on Redo
  • Again, the browser scrolls to the place where the change was made, and we see that that my-number has once more been renamed to another-number
