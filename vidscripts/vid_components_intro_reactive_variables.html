16.1. Connecting input elements to state variables

Top

  • diagram: 2-way connections between input elements and state var
    • this is another entmorph enhancement to HTML,
      • and probably the most important one;
    • [highlight] by setting the :value attribute of an input element to @x
    • [animate] we're effectively connecting that element to the the x state variable;
      • so that when a user interacts with that element,
        • it updates x;
    • Additionally, it also creates a [animate] connection from x back to each element,
      • so that whenever x is updated,
        • ALL elements connected to it update;
  • [show clip of numberbox + range slider updating]
  • This explains why updating one input element also updates the other one,
    • even though we didn't add any Event Handlers;

19. State Variable <=> UI

  • So more generally,
    • including an @Reactive in the :render section
    • let's us update that part of the User Interface whenever the state variable it references updates;
  • and including an @Reactive as an input element's :value attribute
    • let's us update the state variable it references whenever a user interacts with that input element

22. To Replicate in HTML+JS

  • The reason I added the controls to update the circle element's cy and radius properties
    • is to demonstrate how quickly we can add controls to an entmorph component;
  • To create the same thing in HTML and JavaScript,
    • [Zoom in to component so we can highlight better]
    • we would need one event handler for each row in the table;
      • [PAUSE] [highlight each row]
    • we would need to add the event handler to each input element in that row;
      • [PAUSE] [highlight all input elements, in pairs]
    • and we would need each event handler to update the parts of the User Interface where we've added @Reactives in our entmorph component;
      • [PAUSE] [highlight all elements]
  • [HIGHLIGHT REFS] This also means we would have to keep a reference to every element we either want to add an event handler to, or update;
  • That's a lot of code,
    • with a lot of potential to make mistakes,
    • and this is for a relatively simple component;
  • As our User Interface es become more complex,
    • the HTML and JavaScript version will take considerably longer to develop and maintain;
  • Whereas for the entmorph component,
    • we just created 3 state variables,
      • and added an @Reactive
      • to each part of the :render section
      • where we want a state variable
      • to update that part of the User Interface;
  • And we've still managed to avoid creating event handlers;

24. Summary

  • State variables are the central mechanism for updating user interfaces in entmorph,
    • and they provide a declarative approach for building components;
  • Depending on where we place an @Reactive in the :render section
    • we're declaring that we want the state variable it references,
      • to update:
        • an element property,
        • an element child,
      • or that we want an input element to update that state variable;
  • In future videos
    • we'll look at other uses of @Reactives, including:
      • dynamically adding and removing CSS classes,
      • rendering Special Elements;
      • triggering animated effects;
      • and more; "…"
  • We'll also look at more ways to update state variables other than just using input elements;

25. No Virtual DOM

  • In case you're wondering, entmorph doesn't use a Virtual DOM.
  • Instead, it employs a custom-built mechanism based on a Directed Acyclic Graph,
    • to provide its reactive capabilities.
