4. Lament

  • scroll-to :render > :table
  • While copying the row works,
    • if we want to add more controls to control the properties of the circle,
    • we would have to keep copying, pasting and updating the cell label and :value attributes each time we paste;
  • And if we ever want to change the structure of the table,
    • we would have to update all the rows in the table,
    • which would result in a lot of work for us;
  • Rather than use copy and paste, we can avoid all this extra work by moving the whole row into its own component;

5. Add ControlsRow

  • I'll begin by creating a new component;
  • Goto > Cell Top
  • click on Cell Commands > Insert Component Before
  • Id: ControlsRow
  • I'm just going to get rid of the default state variable as this component won't have any state of its own;
    • Clear :state

5.1. Paste tr

  • scroll-to :render
  • Since the last thing we copied was the table row,
    • we can paste it here;
    • and we'll paste it over the root :div element;
  • [open :div for editing]
  • click on Clipboard > Paste
  • [open :tr for editing]
  • If we look at both the input elements,
    • you'll notice that the :value attribute for each element has been reset to nil;
  • This is because when we copied the component,
    • the each :value attribute was set to an @Reactive for the axes-x state variable,
    • which was defined in the :state section of the DestructureComponent;
  • But in this component, there is no axes-x in the :state section;
  • So to avoid any errors,
    • when we pasted the element,
    • Entmorph reset each :value attribute to nil;
  • We need to supply the correct reactive variable to this attribute, which we'll do in a minute;

5.2. Add parameters

  • Our first cell has the label "y-axis (x-coord)",
    • so we need to be able to customize this as well;
  • To provide customization to our ControlsRow component,
    • we'll add parameters to it;
  • Goto > {{{input-params}}}
  • [open Input Parameters for editing]
  • [number-ref]
    • I've used a -ref suffix to indicate that this is going to be a reactive variable;
  • Then I'll add a second parameter,
    • which will be a map of keywords which we'll use in the :render section
    • so I'll click on the Keyword Map Destructure command;
    • For now we'll just expect a :label key, and we'll add more keywords later;
    • [{:keys [label] }]
  • This shows that Input Parameters is another place we can use Destructuring in a Component;
