    • within the Style Rule Commands,

13. Add td:first-child

Top

  • Add Child Rules
  • Add Psuedo-class
    • :first-child
  • This adds a pseudo-class selector of :first-child,
    • notice that the selector has an ampersand prefix;
      • this is a special code introduced by the Sass CSS framework;
      • and it means that the generated selector will have the form:
        • [display] td:first-class
        • so the Style Rule will apply to the first cell of each table row, or tr element;
      • You can also see that the Parent Selector checkbox is checked to indicate this;
      • if we were to uncheck it,
        • the ampersand disappears,
        • and that means that the generated selector will have the form:
          • [display] td :first-class
          • which isn't what we want;
          • click on Parent Selector
  • For the :first-child properties, we'll add:
{:border-top-left-radius
 "30px"
 ;; 
 :border-bottom-left-radius
 "30px"
 ;; Darker than rest of row;
 :background
 "hsl(69, 62%, 40%)"
 ;;
 :color
 "#FFF"
 ;;
 :text-align
 "right"
 }
  • And close the Style Rule

Top

      • remember I had unchecked Parent Selector;

    • Preview Component

    • To understand the difference, let's go up to any row;
      • [scroll-to any tr]
    • Without the parent selector, the :first-child pseudo-class rule would be applied to all descendants of td elements which are the first child of their parent elements;

        • we could only see it for the input/number elements;
    • With the parent selector enabled,
      • the :first-child pseudo-class rule would be applied to all td elements which are the first child of their parent row elements;

24. Set row-radius in :locals

Top

  • Now suppose we want to change both the table and row radii from 30px to 20px;
    • the way we've defined our styles,
    • we would have to change them in 5 places;
      • (once for the table, and once for each individual border radius for the row)
    • if we know that we always want these to be the same, we should be able to set a variable instead;
  • That's what the :locals sub-section of the :styles editor is for;
    • Goto > :styles > :locals
    • [open (:locals) ]
    • click on Add
  • Add a row-radius id
    • clicking on the value opens a CSS Property Value dropdown,
      • meaning that we can create variables for any CSS Property;
    • we'll select String,
    • and set its value to "30px"
