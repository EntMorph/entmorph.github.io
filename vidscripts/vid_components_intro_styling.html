4. Add :styles

Top

  • So let's begin.
  • Go back to first component
  • We'll Goto > Code
  • A component can define its own styles, using the built-in :styles editor;
    • The :styles editor isn't included by default,
      • so we need to add it;
    • To do that, I first have to open the Component editor for editing,
      • [open (Component editor) ]
      • which makes the Component Commands visible;
    • Then I'll click on Add Section
      • and select :styles
  • This adds a :styles editor to the component,
    • which has two subsections,
      • :locals, which we'll look at later (TODO)
      • and :rules, where we can create CSS rules;
  • There's already a default Style Rule, so let's edit that;
    • it has a selector at the top, which has a name ".selector",
      • let's rename it to .main;
        • and press Enter to close the editor
    • Then underneath the selector is a Declaration Block for adding properties;
      • so let's add some properties to it;
      • [open (Declaration Block) ],
{:border      "<open value>"
 ,,,}
  • When I click on the property value, it opens a String by default.
  • If I click on String, it brings up a CSS Property Value editor,
    • which provides dedicated editors for various properties;
    • This can be really useful for visually creating styles, but you always have the option to use strings if you're more comfortable with them;
    • click on cancel
  • 1px solid #AAA
  • Notice that as I add these properties,
    • the Preview panel underneath updates to give you a rough idea of how an element will look, when this Style Rule is applied to it;
{:font-family "verdana"
 :padding     "1em"
 :box-shadow  "0 0 3px 3px #EFEFEF"}
  • I'll collapse the Declaration Block as we won't need to edit it anymore;

13. Add td:first-child

Top

  • Add Child Rules
  • Add Psuedo-class
    • :first-child
  • This adds a pseudo-class selector for :first-child,
    • meaning it will affect the very first element of the parent which the class is applied to;
    • notice that the class has an ampersand prefix;
      • this is a special code used by at least the Sass CSS framework;
      • and it means that :first-child will apply directly to the parent td rule;
      • in other words it will apply to just the first cell of every row;
      • you can also see that the Parent Selector checkbox is checked for this reason;
      • if we were to uncheck it, the ampersand disappears, and that means that this rule will apply to all descendants of td which are the first child of their parents, but not td itself;
      • if this isn't clear now, we'll preview the component in a minute and hopefully it will make more sense then;
  • For the properties, we'll add:
  • Copy background from parent cell for background
  • Paste and decrease lightness to approx. 40

31. Summary

Top

  • Throughout this video we've been using the entmorph component :styles editor to create styles for the component we created in the previous video;
  • The styles editor lets you do all the things a text-based CSS stylesheet can do,
    • but with a number of advantages:
      • We have visual editors which let you add specific blocks such at Style Rules, Child Rules, Declaration blocks and so on, so you don't have to worry about the syntax of any of these entities;
      • You can also use the clipboard to copy and paste any part of your component styles to any other part as long as it's allowed;
      • As you add properties to each Style Rule,
        • you can see a preview of your Style Rule underneath,
        • so you can get a good idea of how your style will look before you your component;
      • A lot of the property values have dedicated editors, and there are many more in development, making it much easier to set a given property value rather than have to remember an obscure code for;
        • Examples include:
          • The browser and CSS Color Editors we looked at
          • Transform
          • … TBD
      • Child Style Rules can be nested inside parent style rules;
        • reflecting the way elements are arranged in the :render section,
        • and making it easier to manage all related styles for a particular element from within one parent Style Rule, rather than having to type out multiple rules like you would in a text editor;
          • For example, we would need to create style rules for all of the following selectors which begin with td
td {...}
td:first-child {...}
td:last-child {...}
td input[type="number"] {...}
td input[type="number"]:focus {...}
  • We looked at how you can add variables to the :locals section,
    • which you can then reference from any style rule property
  • You know how to add CSS classes to elements using two methods:
    • the traditional method, via :class attributes
    • and adding class names to the end of element tag names
  • And when the component is built, it renames the CSS classnames to make them unique to that component, so that you never have to worry about two components having the same class names; they will never clash;
  • Here's an example of it looks
