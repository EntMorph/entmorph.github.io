13. Add td:first-child

Top

  • Add Child Rules
  • Add Psuedo-class
    • :first-child
  • This adds a pseudo-class selector for :first-child,
    • meaning it will affect the first element of the parent which the class is applied to;
    • notice that the selector has an ampersand prefix;
      • this is a special code introduced by the Sass CSS framework;
      • and it means that the generated selector will have the form:
        • [display] td:first-class
        • so the Style Rule will apply to all first cells in each table row;
      • you can also see that the Parent Selector checkbox is checked to indicate this;
      • if we were to uncheck it,
        • the ampersand disappears,
        • and that means that the generated selector will have the form:
          • [display] td :first-class
          • so the Style Rule will apply to all elements which are descendants of td elements and which are the first child of their parent elements,
          • but not the td elements themselves;
  • For the :first-child properties, we'll add:
{:border-top-left-radius
 "30px"
 ;; 
 :border-bottom-left-radius
 "30px"
 ;; Darker than rest of row;
 :background
 "hsl(69, 62%, 40%)"
 ;;
 :color
 "#FFF"
 ;;
 :text-align
 "right"
 }
  • And close the Style Rule

25. Summary

Top

  • The entmorph component styles editor lets you do all the things a text-based CSS stylesheet can do,
    • but with a number of advantages:
      • Most notably, we have visual editors for
        • Style Rule s
          • Selectors
          • Declaration Blocks
      • We also have visual editors for many Property Values, such as
        • Box Shadows
        • Colors
        • Transforms and son on;
        • this means you don't have to worry about remembering the syntax of any of these entities;
        • but if you want to, you always have the option of using Strings;
      • You can also use the clipboard to copy and paste any part of your component styles to any other part as long as it's allowed;
      • As you add properties to each Style Rule,
        • you can see a preview of your Style Rule underneath,
        • so you can get a good idea of how your style will look before you preview your component, something you can't do with text-based stylesheets;
      • Child Style Rules can be nested inside parent style rules;
        • reflecting the way elements are arranged in the :render section,
        • and making it easier to manage all related styles for a particular element from within one ancestor Style Rule,
          • rather than having to type out multiple rules like you would in a text editor;
        • To give you an example, we would need to create style rules for all of the following selectors which begin with td
td {...}
td:first-child {...}
td:last-child {...}
td input[type="number"] {...}
td input[type="number"]:focus {...}
  • and this is just a lot more work to maintain, especially if we need to move things around;
    • We can add variables to the :locals section,
      • which we can then reference from any style rule property
    • When it comes to adding CSS classes to elements, we looked at two methods:
      • the traditional method, via :class attributes
      • and adding class names to the end of element tag names
    • One thing we haven't looked at yet is when the component is built,
      • it renames the CSS classnames to make them unique to that component,
      • so that you never have to worry about two components having the same class names which might clash;
