10. Add table Style Rule

Top

  • Let's style the table next;
  • Goto > Code -> :styles
  • If I click on "Add" in the CSS Rules Commands
    • it's possible to add a new Style Rule or a Keyframes Rule;
  • However, if we go to the .main Style Rule instead;
    • you might have noticed that within the Style Rule Commands,
    • there's an Add Child Rules command button;
    • which when I click on it adds a "Child Rules" section to the .main Style Rule;
  • This lets us add nested Style Rule s within the parent Style Rule;
    • let's add a "table" rule;
    • I'll click on "Style Rule" to add the new rule,
    • and change its selector from .selector to table,
  • Having a table rule nested inside the .main rule means it will only be applied to a table element which is a descendant of an element that has the .main class applied to it;
    • In other words, it will generate a style rule which has a selector of:
      • .main table

 :padding        "0.2em 1.2em"
 :margin-right   "1em"
 }
  • Go back up to .main rule and close everything
  • I'll collapse the table Declaration Block because we're done with it;
  • Let's preview this:
  • Goto > Output > {{{prev-comp}}}
  • So now our table has a grey, rounded background;

13. Add td:first-child

  • This adds a pseudo-class selector for :first-child,
    • meaning it will affect the first element of the parent which the class is applied to;
    • notice that the class has an ampersand prefix;
      • this is a special code introduced by the Sass CSS framework;
      • and it means that :first-child will apply directly to the parent td rule;
      • you can also see that the Parent Selector checkbox to indicate this;

        • and that means that this rule will apply to all descendants of td elements which are the first child of their parents,

22. Summary

Top

  • The entmorph component styles editor lets you do all the things a text-based CSS stylesheet can do,
    • but with a number of advantages:
      • Most notably, we have visual editors for
        • Style Rule s
          • Selectors
          • Declaration Blocks
      • We also have visual editors for many Property Values, such as
        • Box Shadows
        • Colors
        • Transforms and son on;
        • this means you don't have to worry about remembering the syntax of any of these entities;
        • but if you want to, you always have the option of using Strings;
      • You can also use the clipboard to copy and paste any part of your component styles to any other part as long as it's allowed;
      • As you add properties to each Style Rule,
        • you can see a preview of your Style Rule underneath,
        • so you can get a good idea of how your style will look before you preview your component, something you can't do with text-based stylesheets;
      • Child Style Rules can be nested inside parent style rules;
        • reflecting the way elements are arranged in the :render section,
        • and making it easier to manage all related styles for a particular element from within one parent Style Rule, rather than having to type out multiple rules like you would in a text editor;
          • For example, we would need to create style rules for all of the following selectors which begin with td
td {...}
td:first-child {...}
td:last-child {...}
td input[type="number"] {...}
td input[type="number"]:focus {...}
  • and this is just a lot messier to maintain, especially if we need to move things around;
    • We can add variables to the :locals section,
      • which we can then reference from any style rule property
    • When it comes to adding CSS classes to elements, we looked at two methods:
      • the traditional method, via :class attributes
      • and adding class names to the end of element tag names
    • And when the component is built, it renames the CSS classnames to make them unique to that component, so that you never have to worry about two components having the same class names which clash;
    • Here's an example of it looks
    • [show styles under *Preview Component*]
