16. Add td:first-child

Top

  • Add Child Rules
  • Add Psuedo-class
    • :first-child
  • This adds a pseudo-class selector of :first-child,
    • notice that the selector has an ampersand prefix;
      • this is a special code introduced by the Sass CSS framework;
      • and it means that the generated selector will have the form:
        • [display] td:first-class
        • so the Style Rule will apply to the first cell of each table row, or tr element;
      • You can also see that the Parent Selector checkbox is checked to indicate this;
      • if we were to uncheck it,
        • the ampersand disappears,
        • and that means that the generated selector will have the form:
          • [display] td :first-class
          • which isn't what we want;
          • click on Parent Selector
  • For the :first-child properties, we'll add:

27. Set row-radius in :locals

Top

  • Now suppose we want to change both the table and row radii from 30px to 20px;
    • the way we've defined our styles,
    • we would have to change them in 5 places;
      • (once for the table, and once for each individual border radius for the row)
    • if we know that we always want these to be the same,
      • we should be able to set a variable instead;
  • Goto > :styles > :locals
  • That's what the :locals sub-section of the :styles editor is for;
    • it lets us add local variables which we can then reference from our :rules section;
    • [open (:locals) ]
    • click on Add
  • Give it an id of row-radius;
    • clicking on the value opens a CSS Property Value dropdown,
      • meaning that we can create variables which we can then use in any CSS Property;
    • let's select String,
      • and set its value to "20px" as we're changing it;

29. Rendered Stylesheet

  • One more thing I want to show you is when the component is built,
    • any CSS rules you create in the :styles editor should only apply to that component;
    • so any CSS classes which you create in the :styles editor are renamed to make them unique to that component;
  • Here's an example of it looks;
    • I'm going to open the Firefox developer tools and show you the element and class rule for the root div element;
    • Right Click on element in Preview Section
    • Inspect
  • So we can see the div element has a class of "main_" and a string of alphanumeric characters;
    • if we look at the style rule which has been created,
    • it also has the same class name;
    • this makes the .main class unique to our SVGCircleComponent;
  • What this means is that you can have two components with a class name of .main,
    • and you never have to worry about those styles clashing;
