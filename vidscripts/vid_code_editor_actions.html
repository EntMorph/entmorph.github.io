1. Introduction

  • In the previous video, we introduced the Code Editor,
    • where we created code cells for all the data types available in the Namespace Commands panel of the Code Editor;
  • In this video, we're going to look at Actions, which are blocks of code that work with data to produce some output;

2. Create New Project

  • Introducing Code Editor Actions

3. Add my-number

  • We'll start by adding a number;
  • click on Add Data > Number
  • Rename ID: my-number
  • Value: 10

4. Add my-range (range my-number)

  • scroll-to Namespace Commands
  • Add Action > Call
  • Select: range
  • Rename ID: my-range
  • here we're calling the range function, which takes 1 or more arguments;
    • the 1 arity version returns a range of numbers from 0 to the argument;
  • Args: [~my-number~]
  • click on Run
  • Output: (0…9)

5. Add inc-range

  • Add Action > Call
  • Select: map
  • Rename ID: inc-range
  • Args: [ Function > inc, my-range ]
  • Here we're calling the map function, which in this case takes the range of numbers, and calls the inc function to increment each number by 1;
  • click on Run
  • Output: (1...10)
  • We've created 3 cells, one number cell and two action cells, to produce something which is relatively simple;
  • But more than that, we could have done all of this work in just a single cell;

6. Add let Action

  • Goto > {{{ns-comms}}}
  • click on Add Action > let Action
  • Rename Id: my-let-inc-range
  • scroll-to Code Panel
  • there's an Action editor, which contains a let action;
  • Within the let action there are two sections:
    • Bindings, where we add expressions,
    • Body, which returns a single value from bindings;
  • The default let action has an expression with an id of new-item which, is set to nil, and the body returns new-item

6.1. Update Bindings

  • [open Bindings for editing]
  • The Bindings section lets us create any number of expressions, each of which has an id;
  • click on Live Edit
  • TYPE SLOWLY
  • I'll start by renaming new-item -> range-end
    • Notice that as I do this, the reference to new-item in the Body is also renamed;
    • This is another example of automatic reference updating,
    • which was introduced in the last video;
[range-end 10
 my-range (range range-end)
 numbers  (map inc my-range)]
  • [open Body for editing]
    • and we should return numbers instead of range-end in the Body
  • scroll-to Output Panel
  • scroll-to Bindings
  • click on Run
  • Using a let action, we've managed to produce the same result in a single cell that we did earlier using three cells;
  • A let action is very useful for performing a sequence of computations which depend on each other, and returning a single result;

7. if Action

  • Next let's add an if Action
  • click on Cell Commands > Insert Action After > if
  • This creates a new cell containing an if action;
  • This is useful when you need to return one of two alternative values,
    • based on the outcome of a condition;
  • Let's add a condition;
  • click on nil > Action > even?
  • which adds a call to the even? function,
    • which takes one argument;
    • and returns true if the number is even;
  • click on Add > Number
    • [close (Action) ]
  • Let's add both a true and false expression;
  • True: click on nil > String > "Number is even"
  • False: click on nil > String > "Number is odd"
  • scroll-to Run
  • click on Live Edit
  • Now the output shows the string "Number is even"
  • scroll-to even number arg
  • [open Number 0 for editing]
  • click on Commands > +1
  • Rinse and repeat
  • And that's the if action;
    • we don't have to have just strings, we can add any expression in the true or false branches;

8. fn Action

8.1. Add function Action

  • Next let's add a function Action
  • click on Cell Commands > Insert Action After > fn
  • This creates a new cell containing a function action,
  • It has two sections:
    • Parameters
    • Body
  • The Body has a default call action which is asking for a function,
    • so I'll select one now:
    • '*'
  • Rename Id: product
  • We'll add a second parameter;
  • [open Parameters for editing]
    • click on Add Parameter > y
    • [close (Parameters) ]
  • Then within the Body,
    • we'll open the times (or product) Action,
    • Set args: x, y

8.2. Build, Test Function

  • Then, if we scroll-to Run Panel,
  • click on Build Function
  • The Output Panel becomes populated with a Test Function panel,
    • which contains a call to the product function we just created;
  • Not only have we built our function, but we can now test to see how it works by supplying test arguments;
  • e.g. 5, 6
  • And we can do this for other values;
  • If we wanted to, we can add any number of tests by clicking on the Add Test command;

9. Summary

    • let
