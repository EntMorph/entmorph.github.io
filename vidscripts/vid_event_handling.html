7. Add Event Handler to: "-1" Button

Top

  • scroll-to :render > "-1"
  • When a user clicks on this button,
    • it has to subtract 1 from the counter state variable;
  • To do this, we need to add a click event handler to the button;

7.1. Set :on-click to Event Map

  • There's already an :on-click attribute,
    • so if we click on its value,
    • it opens an Event Handler dropdown menu;
  • There's an Event Maps section with quite a few options,
    • a couple of which we'll look at in this video,
    • but for now I'm just going to select Event Map from the very top;
  • This creates an Event Map editor for the :on-click attribute;
  • An Event Map is a declarative mechanism which allows us to configure how an event is handled by an element;
    • The default Event Map contains two keys:
      • :handler
      • and :arg
  • These keys both have a grey background,
    • which means that their values are fixed,
    • and we can't edit them;
  • We need to set :handler to something which is called when the event is triggered;
    • but before we do that we first need to create that something;

7.2. Add {{{uc!}}} setter

  • Goto > Code
  • [open Component for editing]
  • click on Add Section > :setters
  • This creates a :setters section,
    • which allows us to create functions which update state variables;
  • If I click on Add
    • it opens a sub-commands panel with three options:
      • swap!, reset!, and empty;
  • I'll select "swap setter!",
    • click on swap! setter
    • which is the most common one you'll use;
  • This creates a swap! setter;
  • Id: {{{uc!}}}
  • Expr: and it already has an expression,
    • which is a function which takes a single parameter,
    • x
    • and calls the swap! function;
  • [open swap! for editing]
  • The swap! function takes at least two arguments:
    • a reference, which can be a reactive variable;
    • and a function to update that reference;
  • (swap! counter + x)
    • reference to counter
      • I've selected counter and not @counter for the reference;
      • this is because @counter represents the value of counter, whereas the swap! function needs counter without the @ in order to update it;
    • f to Function > +
    • And we need to add one more argument:
      • click on Add > Select Id > x ("the x parameter")
  • scroll-to :setters
  • [close (:setters) ]

7.3. Review

  • We created a setter called {{{uc!}}}
  • Its value is a function which takes an parameter x
    • and calls the swap! function,
      • which updates the counter state variable by adding x to it;
  • Up to now, to update a state variable, we've had to set the :value attribute of an input number or input range element to an @Reactive;
  • So setters provide another mechanism for updating state variables;
  • But, we still need some way to call the setter;

7.4. Add {{{uc!}}} to Event Map

  • So let's do that now;
  • Goto > :render
  • scroll-to "-1" button
  • And within the :on-click Event Map
    • we can now set its :handler, by clicking on nil
    • and selecting Select Setter > {{{uc!}}}
  • and for the :arg,
    • click on nil > Number > -1
  • [close (Event Map) ]

7.5. Review

  • Our -1 Button has an :on-click attribute set to an Event Map,
    • which has a :handler set to {{{uc!}}},
    • and an :arg of -1
  • So when a user clicks on the button, what should happen is that it calls {{{uc!}}} with an argument of -1,
    • which subtracts 1 from counter;

17. Explanation

Top

  • scroll-to input/button with :swap handler
  • The :swap handler is an alternative to creating a setter;
  • It provides exactly the same functionality as {{{uc!}}},
    • but one advantage is we didn't have to call a setter function;
  • The two methods are equivalent to each other and you can use whichever one you prefer;
  • If you need to update the same variable from different places in your User Interface, you probably want to use a setter;
  • Otherwise the :swap handler saves you from having to create a separate setter function;

19. Add Reset button

Top

  • And let's also add a Reset button, which resets the value of counter to 0; 

    [:div
 [:input/button
  {:value "Reset"
   :on-click {:handler :reset
              :arg     {:state counter
                        :value 0}}}]]
  • The :reset handler is very similar to the :swap handler, except that rather than calling a function to update counter, it just sets it to the value of 0 whenever a user presses the Reset button;
