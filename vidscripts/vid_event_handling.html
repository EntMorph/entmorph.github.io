  • Hi, I'm Ali Hussein, creator of Entmorph,
    • and in this video we're going to look at Event Handling in Entmorph Components;
  • So far we haven't had to handle events because we've been using reactive varaibles,
    • which implicitly take care of event handling for us;
  • As long as we work with input elements such as number, range and checkbox,
    • these can update state variables directly;
  • In this video, we're going to introduce buttons, and look at Entmorph's unique features for handling events using Event Maps;

14. Explanation

  • scroll-to input/button with :swap handler
  • So the :swap handler is an alternative to creating a setter;

  • The two methods are equivalent to each other and you can use whichever one you prefer;

15. Event Delegation

  • One thing to note here is that for each new button we add,
    • we have to add an :on-click event handler to it;
    • which quickly becomes cumbersome,
    • not to mention tedious;
  • So let's look at an alternative;

15.1. Delete :on-click attrbutes of all buttons

  • I'm going to start be deleting all of the :on-click attributes for the buttons we've created so far;

15.2. Add :swap Event Map to parent :div

  • scroll-to parent :div
  • click on Add Attributes > Add Event
  • click on on-click
  • For its value
  • click on nil > :swap
  • which creates a :swap handler Event Map;
  • Again, I'll set
    • :state -> counter
    • :fn -> +
    • but for :arg, I'll leave it blank for now;
  • The idea here is that adding an :on-click event handler to the parent div will capture all click events within the div element, including those for each button;

15.3. Set :data-arg attr in each button

  • Next, let's scroll down to the buttons;
  • We can open each button's attributes map starting with the first;
  • click on Add > :data-arg > "-10"
  • click on Add > :data-arg > "-1"
  • click on Add > :data-arg > "1"

15.4. Set :arg-spec in parent div Event Map

  • scroll-to parent div
  • [open Event Map for editing]
  • I'm about to do something which looks a little complex; please bear with me, once I'm done I'll break it down and explain it so hopefully you won't get too confused by what's going on;
  • click on Add
  • which opens a sub-commands panel containing additional keys we can add to the Event Map;
  • we'll cover all of these in due course, but for now I'm going to click on :arg-spec,
    • which adds an Event Map :arg-spec editor;
  • click on Add
  • that opens a sub-commands panel with three keys;
  • click on :data-args,
  • which adds something called an Attribute Coercer Map
  • For the key, I'll type :arg
  • and then when I click on its value, it opens a Select Coercer dropdown;
    • click on :Number
  • scroll-to parent div element
  • [close (div) ]

15.5. Review of :arg-spec

  • in the parent div element, which contains our input buttons,
    • we created an Attributes Map;
  • we added an :on-click attribute,
  • and set its value to a :swap handler Event Map;
  • The Event Map :arg is set to update the counter state variable,
    • by calling the + function on its current value,
    • and passing the value of :arg, which is nil;
  • That isn't actually what we want,
  • so what I'm going to do now is I'm going to delete :arg;
    • [open map editor for editing]
    • select :arg entry
    • click on Delete
  • So our :swap handler is incomplete: it needs to add something to counter whenever a user clicks on a button,
    • but there isn't anything to add any more because we just deleted the :arg key!
  • That's where the :arg-spec key in the Event Map comes in:
    • it provides a specification for creating an argument to pass to our event handler;
  • scroll-to first input/button
  • Remember we set each button's :data-arg attribute to a number,
    • specifically: -10, -1 and 1;
    • so what we'd like to do is to extract that number from the :data-arg attribute, and pass it to the :swap event handler;
  • scroll-to Event Map
  • That's what the :data-args key of the :arg-spec is for;
    • its value is a map of data attribute names we are interested in passing to the event handler, but without the "data-" prefix
    • the fact that we see :arg means we want to extract the value of the :data-arg attribute;
  • But the problem is that attributes are expressed as strings,
    • so we have to convert from strings to numbers;
  • That's why the value of the :arg in the :data-args map is :Number;
    • this says: "convert the value in the :data-arg attribute of the element to a number, before passing it to the :swap handler"
  • If all of this is too much to take in, let's first see this in action;

17. Recap

  • scroll-to Event Map
  • we got rid of the :on-click Event Handler on each button,
  • and instead created a single :swap Event Handler on the parent div element;
  • on each button, we added a :data-arg attribute with the value of that button, as a string
  • and on the :swap Event Map of the parent div, we set the :state and :fn keys of its :arg so to update the counter state variable by adding something to it,
  • and we created an :arg-spec to extract the value of :data-arg from the button we click on, convert it to a number, and pass it to the :swap handler as the :arg key, to replace the :arg key that we deleted;

18. Event Delegation

  • This is a technique know as Event Delegation, and is something which has existed in JavaScript since the beginning;
    • but the Entmorph version I've shown you here requires very little code to produce a lot of functionality which would otherwise require a lot of manipulation of elements;
  • The main advantage is that we only need to create a single event handler to handle different, but related operations, in this case adding different numbers to a state variable
  • Event Delegations works for most types of event handler, including setters; (demonstrate with update-counter!)

21. Event Handlers work directly with state variables

  • In Entmorph, Event Handlers work directly with state variables;
  • unlike traditional event handlers, which usually require working with DOM elements, stopping or preventing events and so on;
