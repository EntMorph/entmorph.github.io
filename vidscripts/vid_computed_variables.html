7. Explanation

Top

  • scroll-to :computed
  • We created twin-x in the :computed section,
    • and set its value to an expression which references x;
  • Since x is a state variable,
    • this means that the value of twin-x at any time depends on x;
  • So whenever x changes, twin-x always has the value of "500 - x";
  • That's why it's called a Computed Variable,
    • because it's value at any time is computed from one, or more, other reactive variables;

8. Comparison: State Variables and Computed Variables

Top

  • Like State Variables, we can add a Computed Variable to most places in the :render section where we want it to update the User Interface;
    • for instance the twin circle's :cx property;
  • The one exception is that we can't update Computed Variables directly using :input elements,
    • so for example we can't set the :value attribute of input number or input range elements to a Computed Variable;
  • And that's the difference between State Variables and Computed Variables;
    • State Variables are read and write,
    • whereas Computed Variables are read-only;
    • The only way to update a Computed Variable, is to update any of the reactive variables it depends on;

14. That's All Folks

Top

  • Goto > :computed
  • So that's all there is to Computed Variables;
  • If you want to update the value of something using an input element,
    • create a state variable and add an @Reactive to it to that input element;
  • If you want to update the value of something based on one or more reactive variables,
    • create a Computed Variable whose value depends on one or more reactive variables;
