2. Introduction

  • Hi, I'm Ali Hussein,
    • and in this video we're going to look at Computed Variables,
    • which are very similar to State Variables which we've already looked at;
  • We'll start from this component,
    • which we've been developing over the last few videos,
  • and we'll end up with this component,
    • which looks identical to our starting component,
    • except for one subtle difference;
  • If we play with the controls in the table,
    • you can see that our circle has now been joined by a second circle;
  • Both circles move in the opposite direction to each other;
  • And if we update the radius of our first circle,
    • our first circle expands,
    • while our second circle contracts by the same amount;
  • So you can think of the second circle as an "opposite twin" of the first circle;

4. Add :computed

  • The second circle has to depend on the first circle for its position and radius,
    • so we can't just create State Variables for it like we did for the first circle;
  • Instead, we need to create Reactive Variables which depend on the Reactive Variables for the first circle;
  • To do that, we need to add a new section for dependant Reactive Variables;
  • Goto > Code
  • [open (Component) ]
  • From the Component Commands
    • click on Add Section > :computed
  • This creates a :computed section,
    • which looks very similar to the :state section;
  • This allows us to create Computed Variables,
    • which I'll do right now to illustrate what they are;
  • click on Add
    • Id: twin-x
    • Expr: click on nil > Action > …
      • this adds an Action editor,
      • with a dropdown menu asking me to Select a Function;
      • I'll select -,
        • which is a function in Entmorph,
      • and this creates a call to the - function;
    • From the Action Args Commands
      • click on Add
      • and change the default Number to 500
      • click on Add
      • Change Editor from Number to x
      • [close (Action) ]
  • Review
    • We've create a computed variable called twin-x
    • its value is an - Action, in this case it means: "500 - x"
    • since our viewport dimensions are 500x500, this makes sure that twin-x has the opposite value to x along the x-axis;

7. Explanation

  • [scroll-to :state]
  • Up to now,
    • we've used State variables,
    • such as x, y and radius
    • to update our User Interface;
  • [scroll-to :render]
  • The way we allow our users to update these State variables is by setting the :value attribute of our input number and input range elements to @x, @y or @radius;
  • [scroll-to :computed]
  • Here we've introduced the twin-x Computed Variable;
    • which as we've seen can also update the User Interface,
    • specifically by setting the twin circle's x position to the opposite position of the original circle;
  • The first thing to note is that we aren't updating twin-x using an input element;
    • at least not directly;
  • Instead, we've set its value to an Action,
    • which returns "500 - x";
  • The key thing here is that x is a state variable,
    • which means that whenever x changes,
    • the value of twin-x also changes;
  • Goto > Output
  • That's why updating x using the X range slider in the table moves the twin circle in the opposite direction of the original circle;
    • because the value of twin-x is always "500 - x";

13. That's All Folks

  • So that's all there is to :computed variables;
  • If you want to update the value of something using an input element,
    • create a state variable and add an @Reactive to it in one or more input elements;
  • If you want to update the value of something using one or more reactive varirables,
    • create a computed variable which depends on one or more state or computed variables,
    • [scroll-to :computed]
    • as we've done here;
