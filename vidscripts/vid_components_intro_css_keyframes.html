3. Add keyframes for border-radius

Top

  • Let's begin;
  • We'll start by adding a Keyframes Rule:
  • Goto > Code > :styles
  • Goto > :rules
  • [open (:rules) ]
  • click on Collapse All
  • click on CSS Style Rules > Add > @Keyframes
    • and this adds a Keyframes Rule editor at the end of our :rules section;
  • At the top it has an Name,
    • which we'll change to radii-animation
  • and underneath that a set of commands,
  • then at the bottom it has a Frames section
    • which is where we add the actual keyframes for our animation;
  • [open (:frames) ]
  • There are already two keyframes by default
    • one at the start, at 0%,
    • and the other at the end, at 100%;
  • Let's add one more frame:
    • click on Add Frame > Empty to add a keyframe with an empty declaration block;
    • Scroll to fit all frames
    • let's scroll back up to see all the frames;
    • notice that although the new frame has been added at the end,
      • the frame which was previously at the end now has a position of 50%,
      • and the new frame is at 100%;
    • this happens automatically, to save you from having to manually re-adjust the positions every time you add more frames;
      • though of course you can edit the position if need to;

10.2. Add animate? state variable

Top

  • Since this is an input element,
    • we can use it to control the value of state variable;
  • In this video we'll use a checkbox to toggle a state variable from true to false;
    • In a future video, we'll look at how a checkbox can add and remove values from a set;

  • Goto > :state

10.3. Set @Reactive(animate?) in "animate?" checkbox

Top

  • Now let's connect our new checkbox to it;
  • click on BrowserBack
  • [scroll-to animate? checkbox]
    • for input number and range elements, we connected the element to a state variable using the :value attribute
    • for checkboxes, we use the :checked attribute instead;
    • open :checked value > @Reactive > animate?
[:tr
 [:td "animate?"]
 [:td [:input/checkbox
       {:checked @animate?}]]]
  • Now whenever a user clicks on this chekbox,
    • it toggles the animate? state variable between true and false;

14. How it works

  • Here's how this works;
  • [scroll-to input/checkbox]
  • The checkbox element has its :checked attribute set to @animate?, which means that when a user clicks on it, it toggles the value from false to true
  • {{{scroll-to(svg > :attrs > :class}}}
  • The svg element has its :class attribute set to a map;
  • This is another entmorph extension to HTML,
    • which makes the :class attribute react to changes in the animate? state variable;
    • the way it works is that when the animate~ state variable switches from false to true,
      • it adds the animate-radii class (the map key) to the <svg> element;
    • Then when the animate? state variable changes back to false
      • it removes animate-radii class from the element;

15. Uncheck Animate?

  • Let's see that in action;
    • uncheck the checkbox,
    • the whole animation abruptly comes to an end;
    • this is because the animate-radii class has been removed from the SVG element,
      • so the element resets to the way it was at the start of the animation;

16. Add Pause? chekbox

Top

  • We can make this look nicer by adding another checkbox to let our users pause and resume the animation;

16.1. Add .pause style

Top

  • First let's add a new Style Rule which will pause the animation when added to an element;
  • Goto > :styles
  • click on Add > Style Rule
    • Selector: .pause
    • properties
{:animation-play-state "paused"}

16.2. Add paused? state var

Top

  • Goto > :state
  • [open (:state) ]
  • click on Add
    • Enter: paused? false

16.3. Add row + checkbox to :render

Top

  • So let's copy the "Animate?" row
  • Paste > update :checked attribute from @animate? to @paused?
[:tr
 [:td "Paused?"]
 [:td [:input/checkbox
       {:checked @paused?}]]]

17. Add @paused? to svg class map

Top

  • Now let's go back to our <svg> element and an entry in the :class attribute map;
  • [scroll-to SVG Element]
  • [open (Attributes) ]
{"pause" @paused?}
  • What this does is that when the @paused? state variable updates, it adds or removes the "pause" class to the <svg> element;
