3. Add keyframes for border-radius

Top

  • We'll start by adding a Keyframes Rule:
  • Goto > Code > :styles
  • click on Collapse All
  • click on CSS Style Rules > Add
    • which opens up a sub-menu;
  • click on @Keyframes
    • to add a Keyframes Rule editor
  • It has an Id,
    • which we'll change to radii-animation
  • and a :frames section
    • which lets us edit the keyframes of our animation;
  • [open (:frames) ]
  • There are two keyframes by default
    • one at the start (0%)
    • and the other at the end (100%) of our animation;
  • Let's add one more frame:
    • click on Add Frame > Empty to add an empty keyframe
    • notice that now the frame stops have been re-arranged to 0%, 50% and 100%;
    • this happens automatically, to save you from having to manually set the stops yourself;

10.1. Add row + checkbox

10.2. Add animate? state variable

Top

  • Since the checkbox is an input element,
    • we need to add an @Reactive to it so that it can control a state variable
  • We first need to add a new state variable, so I'll do that now;
    • Add: [animate? false]
      • we set the initial value to false to indicate that the animation isn't running when we first start the component;

10.3. Set @Reactive(animate?) in "animate?" checkbox

Top

  • Now let's connect the checkbox to it;
  • [scroll-to animate? checkbox]
    • open :checked value > @Reactive > animate?
[:tr
 [:td "animate?"]
 [:td [:input/checkbox
       {:checked @animate?}]]]

11. Create :class map in <svg> element

Top

  • So now we have a chekbox which updates the animate? state variable whenever a user clicks on it;
  • But how do we use that to dynamically add and remove the .animate-radii CSS class?
  • Well, let's go to the <svg> element and do that now;
  • We'll add a :class attribute
    • and for its value, we'll select Map to create an empty map
  • click on Add Entry
    • Key: "animate-radii" (the name of the class we want to trigger)
    • Val: @Reactive > @animate?
  • This is another entmorph extension to HTML;
    • and the way it works is that when the @animate? state variable switches from false to true,
      • it adds the animate-radii class (the map key) to the element;
    • then when the @animate? state variable changes to false
      • it removes animate-radii class from the element;
  • And the way to update the animate? state varaible is by checking the checkbox we added in the table;
  • So let's see this in action;

13. Add Pause? chekbox

Top

  • So let's add a Pause checkbox to let our users pause and resume the animation;

13.1. Add .pause style

Top

  • Goto > :styles
  • click on Add > Style Rule
    • Selector: .pause
    • properties
{:animation-play-state "paused"}

13.2. Add paused? state var

Top

  • Goto > :state
  • [open (:state) ]
  • click on Add
    • Enter: paused? false

13.3. Add row + checkbox to :render

Top

  • So let's copy the "Animate?" row
  • Paste > update :checked attribute from @animate? to @paused?
[:tr
 [:td "paused?"]
 [:td [:input/checkbox
       {:checked @paused?}]]]

Top
